# BLOG_AGENT.md

本文件是本项目的文章写作、审查、发布入口。

处理博客文章内容时，先读 `llms.txt`，再读本文件；需要详细写作、审查、发布规则时继续读 `docs/agent/blog-rules.md`。

代码修改、构建配置、测试或工程结构任务不要从本文件展开；按 `llms.txt` 路由读取 `AGENTS.md`。

---

## 硬规则

1. 文章内容源是 `src/content/posts/*.md` / `*.mdx`。
2. 新文章默认复制 `templates/post.mdx` 后填写，优先使用 `.mdx`。
3. 新增文章后，同步在 `CONTENT_INDEX.md` 增加或更新一行索引。
4. 需要参考旧文章时，先读 `CONTENT_INDEX.md`，只打开 1 到 3 篇最相关正文。
5. `category` 只能是 `AI Agent`、`影刀RPA`、`Skill`、`数据处理`、`视觉实验室`。
6. 新文章默认保持 `draft: true`、`featured: false`、`private: false`。
7. 只有用户明确说“发布”“确认发布”“把 draft 改成 false”“可以公开”等同等意思时，才可改为 `draft: false`。
8. `private: true` 只表示不公开进入站点页面和索引，不是密码保护。
9. 不确定的事实写简短 TODO 或向用户确认，不要编造。
10. 本站是个人博客，不是官方博客或严格工程文档站；Agent 负责整理作者想法，让文章更顺、更适合发布，不要过度干预作者观点。
11. 默认文章形态是视觉笔记：优先用结论、指标卡片、流程图、模块分栏、关键细节和检查清单组织内容。
12. 默认文字风格是专业风格：结构清晰、信息准确、逻辑完整，适合技术文章、项目说明、架构说明、部署说明和开发记录。
13. 大众阅读风格只影响文字表达，不改变视觉笔记这一主体形式。
14. 只有用户明确要求“写给普通人看”“口语一点”“不要太专业”“普通读者能看懂”等，才直接使用大众阅读风格；如果 Agent 判断主题明显更适合大众阅读，先提醒用户确认，不要直接改风格。
15. 事实、严谨性或政治相关内容只做轻量提醒：发现明显错误可标注或简短建议，不要把文章改成没有个人味道的安全稿。
16. 引用可下载文件时使用 `/files/` 开头链接；`public/files/` 都是公开文件，不能放私密资料；private 文章不等于 private 文件。
17. 发布前必须检查敏感信息、模板占位词，并运行必要验证。

## 详细规则

详细文章规则见：

- `docs/agent/blog-rules.md`：文章来源、frontmatter、写作结构、视觉组件、图表、审查、发布、静态文件。
- `docs/agent/blog-skills/`：写作 brief、研究、结构、审稿 prompt、Skill 化判断等参考库；普通文章任务不默认通读。

当本文件和 `docs/agent/blog-rules.md` 都适用时，以更具体、更严格的规则为准。
