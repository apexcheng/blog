## 对比表

简单对比继续用 Markdown 表格，不必为了视觉效果强行写组件。

| 写法 | 适合场景 | 注意事项 |
| --- | --- | --- |
| Markdown 段落 | 普通叙述、经验记录、解释性正文 | 最稳定，也最容易维护 |
| Markdown 表格 | 轻量对比、参数说明、规则清单 | 内容不要太长 |
| MDX 组件 | 指标、流程、重点结论、模块拆解 | 只在结构真的需要被看见时使用 |

## 总结块

<HighlightBox eyebrow="CHECK" title="MDX 只做必要的可视化">
  写文章时先用 Markdown 把意思说明白，再用 MDX 强化阅读路径。组件越少，读者越容易看见真正重要的内容。
</HighlightBox>
